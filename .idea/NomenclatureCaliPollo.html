Estándares de Nomenclatura

Este documento establece las convenciones de nomenclatura para clases, métodos y comentarios en el proyecto Cali Pollo. Seguir estas pautas es esencial para mantener la consistencia y legibilidad del código.

Nomenclatura en HTML y CSS

Clases en HTML/CSS

Para el proyecto Cali Pollo, utilizaremos la metodología BEM (Block, Element, Modifier) para nombrar nuestras clases CSS:

/* Bloque: Representa el componente principal */
.menu-item { }

/* Elemento: Parte del bloque que no puede existir independientemente */
.menu-item__title { }
.menu-item__price { }
.menu-item__description { }

/* Modificador: Variante o estado del bloque */
.menu-item--featured { }
.menu-item--sold-out { }

Reglas específicas:

  • Nombres de clase en minúsculas
  • Palabras separadas por guiones (-)
  • Elementos con doble guion bajo (__)
  • Modificadores con doble guion (--)
  • Nombres descriptivos relacionados con la función, no con la apariencia

Ejemplos específicos para Cali Pollo:


<section class="menu">
    <div class="menu-item">
        <h3 class="menu-item__title">Pollo a la Parrilla</h3>
        <p class="menu-item__description">Pollo marinado en especias</p>
        <span class="menu-item__price">$15.000</span>
    </div>

    <div class="menu-item menu-item--featured">
        <h3 class="menu-item__title">Pollo Asado Caleño</h3>
        <p class="menu-item__description">Especialidad de la casa</p>
        <span class="menu-item__price">$18.000</span>
    </div>
</section>

IDs en HTML

Los IDs deben usarse exclusivamente para:

  • Referencias de anclas en la navegación
  • Conectar etiquetas con elementos de formulario
  • JavaScript cuando es necesario identificar un elemento único

Reglas para IDs:

  • Usar camelCase
  • Ser específicos y descriptivos
  • No usar para estilos (preferir clases)
<section id="menuPrincipal">
<form id="formularioContacto">
<div id="mapaUbicacion">

Nomenclatura en JavaScript

Clases

Las clases en JavaScript usarán PascalCase:

class MenuItem {
    constructor(name, price, description) {
        this.name = name;
        this.price = price;
        this.description = description;
    }
}

    // ...
}

class OrderManager {
    // ...
}

Métodos y Funciones

Los métodos y funciones usarán camelCase y deberán ser verbos o frases verbales:

// Métodos dentro de una clase
class MenuManager {
    addMenuItem(item) {
        // ...
    }

    calculateTotal() {
        // ...
    }

    displayFeaturedItems() {
        // ...
    }
}

// Funciones independientes
function validateOrder(order) {
    // ...
}

function formatCurrency(amount) {
    // ...
}

// Funciones anónimas y arrow functions
const showNotification = (message) => {
    // ...
};

Prefijos recomendados:

  • get... - Para obtener un valor
  • set... - Para establecer un valor
  • is... - Para verificaciones booleanas
  • has... - Para verificar existencia
  • calculate... - Para cálculos

Interfaces y Traits

Las interfaces llevarán el sufijo "Interface" y los traits llevarán nombres descriptivos de su funcionalidad:

<?php
namespace CaliPollo\Menu;

interface MenuItemInterface
{
    public function getPrice();
    public function getName();
}

trait Discountable
{
    private $discountRate;

    public function applyDiscount($amount)
    {
        return $amount * (1 - $this->discountRate);
    }
}

Comentarios en JavaScript

Comentarios de función:

/**
 * Calcula el precio total de los ítems en el carrito
 * aplicando descuentos si corresponde
 *
 * @param {Array} items - Array de objetos MenuItem
 * @param {Number} discountRate - Tasa de descuento (0-1)
 * @return {Number} El precio total calculado
 */
function calculateCartTotal(items, discountRate = 0) {
    // Verificamos que los items sean válidos
    if (!Array.isArray(items)) {
        throw new Error('Items debe ser un array');
    }

    // Sumamos el precio de cada ítem
    let total = 0;
    for (const item of items) {
        total += item.price;
    }

    // Aplicamos descuento si existe
    if (discountRate > 0) {
        total = total * (1 - discountRate);
    }

    return total;
}

Comentarios en PHP

<?php
namespace CaliPollo\Order;

/**
 * Clase para procesar pedidos del restaurante
 *
 * Esta clase maneja la lógica de procesamiento de pedidos,
 * incluyendo validación, cálculo de precios y preparación
 * para envío a cocina.
 *
 * @author Equipo Cali Pollo
 */
class OrderProcessor
{
    /**
     * Procesa un nuevo pedido
     *
     * @param array $orderData Datos del pedido
     * @param int $customerId ID del cliente
     * @return Order Objeto pedido procesado
     * @throws InvalidOrderException Si el pedido es inválido
     */
    public function processOrder(array $orderData, int $customerId): Order
    {
        // Validar datos del pedido
        $this->validateOrder($orderData);

        // Crear nuevo objeto de pedido
        $order = new Order($customerId);

        // Añadir ítems al pedido
        foreach ($orderData['items'] as $item) {
            $order->addItem($item['id'], $item['quantity']);
        }

        return $order;
    }
}
