Introducción a los Estándares de Codificación

Este documento establece los estándares de codificación para el desarrollo del proyecto web Cali Pollo. Seguir estos estándares asegura una base de código consistente, mantenible y de alta calidad.

Estándares HTML

Estructura Básica

Utilizar HTML5 con la estructura adecuada incluyendo DOCTYPE, html, head y body.

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Cali Pollo - Título de la Página</title>
</head>
<body>
    <!-- Contenido principal -->
</body>
</html>

Indentación y Formato

  • Usar 4 espacios para la indentación, no tabuladores
  • Elementos anidados deben estar indentados adecuadamente
  • Nombres de etiquetas y atributos en minúsculas
  • Utilizar comillas dobles para los valores de los atributos

Organización

  • Separar estructura (HTML), presentación (CSS) y comportamiento (JavaScript)
  • Utilizar etiquetas semánticas (header, nav, main, section, article, footer, etc.)
  • Incluir atributos alt en imágenes para accesibilidad

Estándares CSS

Estructura y Organización

  • Archivos CSS externos para separar contenido de presentación
  • Organizar reglas por componentes o secciones
  • Comentar secciones importantes
/* ===== HEADER STYLES ===== */
header {
    background-color: #ffcc00;
    padding: 20px;
}

/* ===== NAVIGATION STYLES ===== */
nav {
    display: flex;
    justify-content: center;
}

Formato

  • Una propiedad por línea
  • Espacio después de los dos puntos en las propiedades
  • Punto y coma después de cada declaración
  • Usar clases en lugar de IDs cuando sea posible

Nomenclatura BEM

Utilizar la metodología BEM (Block, Element, Modifier) para nombrar clases:

.menu-item { }
.menu-item__price { }
.menu-item--featured { }

Estándares JavaScript

Sintaxis

  • Usar ECMAScript moderno (ES6+) cuando sea posible
  • Punto y coma al final de cada declaración
  • Preferir const sobre let; evitar var
// Correcto
const menuItems = document.querySelectorAll('.menu-item');
let totalPrice = 0;

// Incorrecto
var menuItems = document.querySelectorAll('.menu-item')

Estilo y Formato

  • Indentación de 2 espacios
  • Líneas de máximo 80 caracteres
  • Usar camelCase para variables y funciones
  • Usar PascalCase para clases
  • Espacios alrededor de operadores

Buenas Prácticas

  • Evitar variables globales
  • Usar módulos para organizar el código
  • Comentar funciones complejas
  • Manejo adecuado de errores

Estándares PHP

Estructura y Formato

  • Utilizar PHP 7.4+ y sus características
  • Tags PHP: usar siempre <?php ?>, nunca tags cortos
  • Indentación de 4 espacios
  • Llaves en nueva línea para clases y funciones, misma línea para condicionales y bucles
<?php
namespace CaliPollo\Menu;

class MenuItem
{
    private string $name;
    private float $price;

    public function __construct(string $name, float $price)
    {
        $this->name = $name;
        $this->price = $price;

        if ($price <= 0) {
            throw new \InvalidArgumentException("El precio debe ser mayor que cero");
        }
    }
}

Nomenclatura

  • CamelCase para métodos y variables
  • PascalCase para clases
  • UPPER_SNAKE_CASE para constantes
  • Nombres descriptivos y en español para mejor comprensión

Seguridad

  • Sanitizar todas las entradas de usuario
  • Usar consultas parametrizadas para bases de datos
  • Validar datos antes de procesarlos
  • Gestionar adecuadamente las sesiones
